内容填充指南

站点框架已完成，本指南说明协作者如何向每个专题页填充技术分析内容。

1. 本地预览

本站点是纯静态站点，所有数据已内联到 assets/js/site-data.js，无需任何服务进程。

  • 直接双击 index.html 即可在浏览器中查看。
  • 也可部署到 GitHub Pages 等静态托管：把项目目录推送到仓库的 Pages 分支或子目录即可。
  • 可选：本地仍可起 python -m http.server 8000 --directory .，但不是必须。

2. 数据更新

真实数据保存在 assets/data/*.json。修改后请运行以下脚本重新生成 assets/js/site-data.js：

python scripts/bundle-data.py

该脚本会读取 topics.json、learning-paths.json、code-map.json 并写入 JS 全局变量 window.SITE_DATA。

3. 新增专题

  1. 复制 assets/templates/topic-page.html 到 pages/<分类>/<topic>.html。
  2. 编辑 assets/data/topics.json，添加专题元信息。
  3. 必要时在 assets/data/code-map.json 增加源码索引。
  4. 必要时新建 pages/<分类>/ 子目录。

4. 专题页填充清单

  • 更新 hero 描述和阅读时间。
  • 补充“本页要回答的问题”里每一个问题的答案。
  • 在交互区接入流程图或 step JSON 动画。
  • 在源码入口列出真实代码引用，使用 path:line 格式。
  • 状态从 todo / outline 升级到 draft / ready。

5. 状态规范

  • todo：尚未开始。
  • outline：已列出结构和问题清单。
  • draft：内容已写但尚需 review。
  • ready：内容可发布。
  • advanced：高级专题，默认不在入门路径中。

6. 目录与分类

  • pages/core/：架构、调度等核心运行链路。
  • pages/distributed/：进程通信、并行、KV 传输。
  • pages/decoding/：投机解码、采样等解码算法。
  • pages/optimizations/：KV cache、prefix caching、量化等性能专题。
  • pages/models/：各模型结构专题。
  • pages/reference/：术语表、内容填充指南。
